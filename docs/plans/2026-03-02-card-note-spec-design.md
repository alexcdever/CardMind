# 卡片笔记规格设计

- 日期：2026-03-02
- 状态：approved
- 目标文件：`docs/specs/card-note.md`

## 1. 背景

为补齐 CardMind 正式规格体系，需要将卡片笔记领域从历史计划文档和实现细节中抽离，形成语言无关、可观察行为导向的正式规格文档。

## 2. 目标

1. 定义卡片笔记领域的生命周期、显示规则、搜索规则、筛选与排序规则。
2. 明确与数据池元数据的关联边界（自动挂接与去重）。
3. 明确协作可见性与最终一致性约束。
4. 提供黑盒验收标准，支持规格先于实现演进。

## 3. 非目标

1. 不定义数据库表结构、索引实现、搜索引擎实现。
2. 不定义底层同步或冲突合并算法。
3. 不定义 UI 具体组件、布局与交互动效。

## 4. 关键决策

1. 生命周期包含创建、编辑、软删除、恢复；当前版本不提供用户侧恢复入口。
2. 列表固定只显示未删除笔记；不提供删除状态筛选给用户。
3. 搜索仅在未删除笔记范围执行，规则为标题+正文子串匹配且不区分大小写。
4. 排序支持更新时间与创建时间，默认按更新时间倒序。
5. 创建或加入数据池时自动挂接已有笔记 `noteId` 到池元数据，且包含已软删除笔记。
6. 已挂接笔记后续编辑/软删除/恢复不新增重复 `noteId` 引用。

## 5. 文档结构

1. 使用说明（AI/实现者）

6. 数据池关联与协作一致性
7. 错误语义与验收
8. 兼容与演进
