# 领域共享契约规格设计

- 日期：2026-03-02
- 状态：approved
- 目标文件：`docs/specs/shared-domain-contract.md`

## 1. 背景

项目正在从“计划文档驱动实现”转向“正式规格驱动实现”。为了避免分域规格（卡片笔记、数据池、本地设置、UI 交互）在术语、错误语义、验收口径上出现漂移，需要先定义一份跨域共享契约作为总则。

## 2. 目标

1. 统一分域规格的术语、规则强度与判定边界。
2. 定义跨域一致性、幂等、权限、错误语义与验收契约。
3. 明确“规格先于实现”的变更顺序。
4. 保持文档语言无关，不绑定具体技术实现。

## 3. 非目标

1. 不定义某个分域的具体业务流程细节。
2. 不定义底层存储、协议、冲突合并算法细节。
3. 不替代分域规格，仅提供分域必须遵守的共享规则。

## 4. 核心决策

1. 文档作为跨域“约束上层”，分域规格可细化但不可冲突。
2. 采用 MUST/SHOULD/MAY 作为统一规则强度。
3. 统一黑盒验收原则：以可观察行为为准，不依赖实现细节。
4. 并发只约束结果一致性与可解释性，不约束具体算法。

## 5. 文档结构

1. 使用说明（给 AI/实现者）
2. 目的与适用范围
3. 统一术语与规则强度
4. 跨域一致性、权限、错误、验收契约
5. 与分域规格关系与演进规则

## 6. 交付与影响

1. 新增 `docs/specs/shared-domain-contract.md`。
2. 更新 `docs/specs/DIR.md` 索引。
3. 本文档为后续补齐 `card-note`、`local-settings`、`ui-interaction` 规格提供统一约束基线。
