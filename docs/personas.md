# 用户画像

> 本文档记录 CardMind 的目标用户画像与核心使用场景，约束所有产品体验判断。
> 受 `docs/product.md` 约束，是其下游产品文档。

---

## P1：Alex（核心画像）

> 此画像是本项目发起人的真实使用场景提炼。

### 基本信息

| 维度 | 描述 |
|------|------|
| 身份 | 技术背景的个人用户，非团队管理者 |
| 设备组合 | Android 手机（主力查看）、Windows 台式机（主力写作）、MacBook Air（辅助） |
| 使用频率 | 手机：每天白天高频查看、低频记录 / 桌面端：工作日集中写作 |
| 使用场景 | 通勤路上记录一闪而过的想法 / 工作间隙整理成笔记 / 回家后台式机上续写 |

### 当前工具与痛点

| 维度 | 现状 |
|------|------|
| 当前工具 | 小米内置备忘录 |
| 最大痛点 | 备忘录"不纯粹"——打开一个笔记 app 却看到思维导图推荐、云服务容量警告等无关内容 |
| Markdown 体验 | 内置备忘录 Markdown 支持不完整，中文半角符号场景下几乎不可用 |
| 内容组织 | 仅有文件夹归类，没有标签系统，回头找笔记困难 |
| 同步现状 | 手机厂商的云服务与照片/通讯录等捆绑在一起，容量被照片占满，不敢用来同步笔记 |
| 为什么不做 Obsidian | 多设备同步需要额外配置，学习成本高于实际需求 |
| 为什么不做 Notion | 必须联网才能同步，不满足局域网场景 |

### 期望的产品体验

1. **打开即写**：从点击图标到开始打字 <= 2 秒，与系统备忘录一致
2. **干净无干扰**：没有推荐、没有提示、没有捆绑服务，只有编辑器和内容
3. **完整 Markdown**：标题、粗体、列表、代码块等在中英文混排下稳定可用
4. **标签系统**：按标签归类而非仅靠文件夹，方便有明确目的时回去翻查
5. **局域网同步**：手机写的笔记在关机前自动出现在电脑上，无需手动操作，不依赖外网和云容量
6. **跨平台**：同一套数据在 Android / Windows / macOS 间延续使用

### 使用习惯

- **写完之后会回看**，但频率低于写——总是在"需要找某条笔记时"才打开回顾，而不是闲着没事翻
- **情绪和想法类笔记**为主，不是知识管理体系，不需要复杂组织结构
- 手机端偏**查阅**，桌面端偏**创作**

---

## 画像使用规则

1. 所有产品体验判断 MUST 优先以 P1（Alex）的使用场景为出发点，而非泛化的"用户可能……"
2. 当新增功能不能使 Alex 在他当前的场景中受益时，SHOULD 视为偏离当前产品阶段
3. 当多个方向冲突时，优先级排序依据 Alex 的痛点严重程度而非功能吸引力
