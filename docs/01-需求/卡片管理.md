# 卡片管理需求文档

## 1. 概述

本文档描述了CardMind应用中卡片管理相关的功能需求，包括卡片的创建、编辑、查看、删除、排序与显示等功能。

## 2. 功能需求

### 2.1 卡片基本操作

#### 2.1.1 创建卡片
- 用户可以创建新卡片，包含标题和内容
- 标题限制：最多20个字符
- 内容限制：最多2000个字符
- 创建时间精度：毫秒级，自动生成
- 修改时间精度：毫秒级，自动更新

#### 2.1.2 编辑卡片
- 用户可以编辑已存在的卡片
- 编辑时保留原创建时间，仅更新修改时间
- 实时显示标题和内容的字数统计

#### 2.1.3 查看卡片
- 用户可以查看卡片的完整信息
- 卡片详情显示：完整标题、完整内容、创建时间和修改时间

#### 2.1.4 删除卡片
- 用户可以删除卡片
- 删除前需要二次确认
- 删除后无法撤销
- 支持软删除，标记卡片为已删除状态

### 2.2 卡片排序与显示

#### 2.2.1 排序规则
- 默认按修改时间倒序排序
- 相同修改时间按标题字母顺序排序

#### 2.2.2 卡片列表显示
- 卡片列表中每张卡片显示：部分标题、部分内容、创建时间和修改时间
- 卡片数量：无限制
- 空列表提示：当卡片列表为空时显示友好提示

#### 2.2.3 删除操作交互
- 在卡片列表中左滑或长按出现删除按钮
- 点击删除按钮后显示二次确认弹窗

### 2.3 卡片数据同步

- 支持在多个设备间同步卡片数据
- 使用Yjs的CRDT算法实现无冲突数据合并
- 支持离线使用，数据本地存储

## 3. 注意事项

- 卡片内容有字数限制
- 暂不支持撤销删除操作
- 卡片数据默认在本地加密存储
