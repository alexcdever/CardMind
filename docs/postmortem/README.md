# Postmortem 复盘

> 记录踩过的坑和解决方案，避免重复犯错
> 每次 `/postmortem` 后创建新文件

## 文件命名格式

`YYYY-MM-DD-简短描述.md`

例如：
- `2024-03-15-frb-binding-sync-error.md`
- `2024-03-10-test-flakiness.md`

## 模板

```markdown
# Postmortem: [标题]

**日期**: YYYY-MM-DD  
**发现者**: [你的名字]  
**严重程度**: [低/中/高/严重]

## 问题描述

发生了什么？

## 时间线

- `14:30` - 发现问题
- `14:35` - 尝试方案 A
- `14:50` - 发现方案 A 无效
- `15:10` - 找到根因
- `15:30` - 解决问题

## 根因分析

### 表面原因

看起来是因为...

### 真正原因

实际上是因为...

## 解决方案

如何解决的？

## 预防措施

如何避免再次发生？

## 教训总结

- 教训 1
- 教训 2

## 是否需要写入全局记忆？

- [ ] 是 - 写入 ~/.agents/memory/
- [ ] 否 - 仅项目级
```
