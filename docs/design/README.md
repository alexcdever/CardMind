## 📋 关于本目录

本目录包含 CardMind 的**视觉设计规范**，定义应用的外观、颜色、字体、布局等设计元素。

**重要区分**：

- ✅ **`openspec/specs/features/`**：交互规格（用户操作、状态变化等）- **可执行测试**

## 🎨 设计原则

CardMind 的设计遵循四大原则：

1. **简洁专注** - 最小化视觉干扰，让内容成为焦点
2. **快速响应** - 清晰的视觉层级，快速定位信息
3. **轻量舒适** - 柔和的颜色，充足的留白，支持深色模式
4. **一致可靠** - 统一的设计语言，可预测的交互模式

## ⚠️ 重要提醒

### 设计 vs 规格

| 维度 | 设计文档（本目录） | 交互规格（openspec/specs/） |
|------|------------------|---------------------------|
| **内容** | 颜色、字体、布局、间距 | 用户操作、状态变化、行为约束 |
| **可执行性** | ❌ 不可执行 | ✅ 可通过测试验证 |
| **示例** | "按钮是蓝色的" | "点击按钮后显示加载状态" |
| **工具** | Figma、设计稿 | Flutter 测试、Spec Coding |

### 文档更新原则

- **设计文档**：视觉元素变更时更新（颜色、字体、布局等）
- **交互规格**：交互行为变更时更新，并同步更新测试用例

2. 在 `component_library.md` 添加视觉规范
3. 在 `openspec/specs/features/` 创建交互规格（使用 OpenSpec 流程）
4. 实现组件并编写测试
