# Docs DIR 索引说明

## 目标

- 约束 `docs/` 及其关键子目录中的索引文件维护方式
- 帮助人和 AI 快速理解文档目录职责与阅读顺序
- 避免把目录导航规则扩张成高维护成本的流程门禁

## 适用范围

- 本文档仅适用于 `docs/` 目录及其关键子目录
- 代码目录、平台目录、资源目录与生成目录默认不使用 `DIR.md`

## 当前建议

- 当 `docs/` 下的关键目录新增、删除、重命名文档时，应同步更新对应 `DIR.md`
- `DIR.md` 仅承担目录索引与简要定位说明，不作为中心治理机制
- 是否维护代码文件头、额外元数据或脚本化校验，由具体任务与实际收益决定，不再作为仓库级强制要求

## 非目标

- 本文档不强制所有目录都维护 `DIR.md`
- 本文档不要求每次代码改动都同步维护文件头元数据
- 本文档不定义额外门禁脚本
