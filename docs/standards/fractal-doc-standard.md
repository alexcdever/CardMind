# 分形文档规范

## 强制性规则

- 任何功能、架构或实现的变更**必须**更新受影响目录中的 `DIR.md` 条目。
- 每个需要维护的目录**必须**包含 `DIR.md`，包括适用的仓库根目录。
- `DIR.md`**必须**以简洁的定位文本开头（不超过三行），然后列出文件名、角色和职责。
- 当修改 `.dart` 或 `.rs` 文件时，其三行文件头**必须**保持真实并与目录索引同步。
- `.dart` 和 `.rs` 文件的源文件头**应该**严格使用以下标准键：
  - `input:` - 输入
  - `output:` - 输出
  - `pos:` - 定位/职责

## 范围排除

以下生成产物和第三方依赖不在此规范范围内：
`build/`、`rust/target/`、`ios/Pods/`、`android/.gradle/`、`linux/build/`、`macos/Build/`、`windows/build/`、`pubspec.lock`、`lib/**.g.dart`、`lib/**.freezed.dart`

## 执行指南

- 本规范是策略驱动的。合规性**必须**在审查和交付流程中强制执行，而非通过仓库门禁脚本。
