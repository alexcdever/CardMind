# UI 风格规范

本文档定义 CardMind 的 UI 风格基线，以 Digital Parchment 设计系统为准，作为页面设计、组件实现与原型复现的统一参考。

## 设计方向

CardMind 的设计语言以 **Digital Paper**（数字纸张）为核心理念——优先保障内容持久性、本地优先可信度和认知专注度。面向追求速度与清晰度的用户，避免装饰性元素。

美学风格为 **极简** 与 **功能驱动**，从精密工具和技术文档中汲取灵感。通过微妙的边框和有意的留白建立视觉层次，而非使用重阴影或鲜艳渐变。界面应表现为高性能、低延迟的思维延伸——稳健、安静、可靠。

## 色彩

色板建立在 **Paper and Ink**（纸墨）基础上。

- `paper-white` 是主画布色，比纯 `#FFFFFF` 更柔和，减少眼部疲劳
- `ink-charcoal` 是正文色，确保最大对比度和可读性
- `active-teal` 作为唯一强调色，克制使用于焦点、主操作和活跃导航态
- `muted-border` 用于所有结构分隔，维持纸张隐喻的一致性

完整色板见 `docs/standards/design-tokens.md`。

## 字体排版

采用双字体策略：

- **Inter**：UI 元素主力字体，中性系统的感觉，融入背景而不喧宾夺主
- **JetBrains Mono**：笔记编辑器专用，强调纯文本和本地优先性质

层次通过字重（SemiBold）和颜色（Charcoal vs Gray）建立，而非大幅字号变化，保持界面紧凑。

字号层级见 `docs/standards/design-tokens.md`。

## 布局与间距

布局遵循严格的 **固定面板**（桌面端）和 **堆叠**（移动端）模型。

- **桌面端（三栏）**：侧栏 260px \| 笔记列表 360px \| 编辑器（流式，最大宽 48rem 居中）
- **移动端（双 Tab）**：底部导航栏恰好两个入口：「卡片」和「数据池」
- **网格**：一致的 4px/8px 增量网格，所有内边距和外边距必须是 4 的倍数
- **标准容器内边距**：16px
- **紧凑列表项间距**：8px

## 层级与深度

通过 **色调分层** 和 **微妙轮廓** 表达深度，不使用阴影。

- **Level 0（背景）**：应用主框架，使用基础中性色
- **Level 1（表面）**：侧栏等次层面板，比主画布深/浅 1-2%
- **边框**：1px 实线 `muted-border` 定义卡片、面板和输入框边界
- **激活态**：选中项添加 2px 左描边 `active-teal`，不改变元素层级

## 形状

形状语言 **柔和但有纪律**。

- **标准圆角**：4px——比直角更现代，比高度圆角更严肃
- **交互元素**：按钮和标签保持同一 4px 圆角
- **图标**：线性描边图标（1.5px 或 2px 描边），匹配边框为主的美学

## 组件

### 按钮
- 扁平表面 + `muted-border` 边框
- 主按钮：`active-teal` 背景 + 白色文字
- 次按钮：白色背景 + 青色文字/边框
- 无阴影

### 笔记卡片
- 矩形容器 + 底部边框分隔线
- 悬停/激活时背景变冷灰，或添加青色左强调线

### 输入框
- 极简设计，仅底部 2px 边框
- 聚焦时边框变 `active-teal`
- 搜索栏可使用完整边框容器

### 数据池卡片
- 大尺寸居中卡片（未加入态）
- 1px 边框 + 宽松内边距（24px）

### 确认对话框
- 危险操作（删除/解散）使用高对比模态框
- 确认按钮使用 `danger-red` 强调

### 同步指示器
- 小型文字标签，位于侧栏或底栏
- 例："• Synced" 或 "• 2 Pending"
- 使用 `status-sm` 字号

## 设计令牌

具体 Token 值（色值、圆角、间距、字号）见 `docs/standards/design-tokens.md`。

## 与规格文档的边界

- 本文档是长期 UI 风格规范，不定义页面行为真相
- 具体页面布局、状态流转、交互规则由代码与测试定义
- 若实现与本文档冲突，以用户要求为准

> 本文档以 Digital Parchment 设计系统为基准。
