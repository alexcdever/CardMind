## 遇到的问题及解决

### 问题 1: LoroValue 构造
- **现象**: `LoroValue::List(Box::new(vec![]))` 编译失败
- **原因**: loro 1.10.0 要求使用 `LoroListValue(Arc<Vec<LoroValue>>)`
- **解决**: 使用 `vec![].into()` 自动转换

### 问题 2: EndpointAddr 类型限制
- **现象**: `EndpointAddr` 未实现 `Display` trait
- **解决**: 使用 `format!("{:?}", addr)` 代替 `to_string()`

### 问题 3: PoolNetwork 测试依赖复杂
- **现象**: PoolNetwork 需要 PoolEndpoint、PoolStore、CardNoteRepository
- **解决**: 创建辅助函数 `create_test_pool_network()` 在测试中初始化所有依赖

### 问题 4: 边界扫描器 Rust 覆盖率显示 0%
- **现象**: 334 个 Rust 测试通过，但扫描器显示 Rust 0% 覆盖
- **根因**: 扫描器只读取 `coverage/lcov.info`（Flutter），未读取 Rust LCOV
- **解决**: 
  1. 配置 `cargo tarpaulin --out Lcov` 生成 `rust/lcov.info`
  2. 修改扫描器同时读取两个 LCOV 文件
  3. 修改路径匹配逻辑支持绝对路径（Rust LCOV 使用绝对路径）

### 问题 5: Flutter 质量门禁被 Rust 未覆盖边界阻塞
- **现象**: `dart run tool/quality.dart flutter` 即使 Flutter 覆盖率已达标，仍因 Rust 高优先级边界退出非零
- **根因**: `tool/test_boundary_scanner.dart` 统一扫描 Flutter + Rust，但 `quality.dart flutter` 未限制 gate 范围
- **解决**:
  1. 扫描器新增 `--scope=flutter|rust|all`
  2. `quality.dart flutter` 使用 `--scope=flutter`
  3. `quality.dart rust` 使用 `--scope=rust`

### 问题 6: 边界扫描器对 Flutter UI 误报过多
- **现象**: 普通命名参数、字段声明、`?? SizedBox.shrink()` 等被统计为高优先级边界
- **根因**: Dart AST 规则过于偏语法扫描，没有区分“有测试价值的边界”和“低价值 UI 配置噪音”
- **解决**: 增加过滤规则，保留真实交互/异常路径，忽略低价值 UI 噪音
