## 遇到的问题

- `tool/build.dart run --dart-define ...` 启动后的 macOS app 在写仓库 `tmp/` 路径时没有稳定产出调试文件
  - 处理：改为从 app 容器目录读取 `debug_status.log` 与 `debug_invite.txt`
- `macOS owner -> iOS joiner` 与 `iOS owner -> macOS joiner` 两条真实路径都在连接建立阶段超时
  - 现状：两端都能启动并初始化 runtime，但最终都收敛到 `connect failed: internal error: timed out`
- macOS app 从隐藏 worktree 路径直接读取外部运行态 dylib 时会报 `file system sandbox blocked open()`
  - 处理：把 `build/native/macos/libcardmind_rust.dylib` 复制到 app bundle `Contents/Frameworks/` 后，真实 `flutter run -d macos` 验证恢复可用
- macOS Runner 原先缺少 `com.apple.security.network.client`
  - 现状：修复前 owner/joiner 真实路径稳定卡在 connect 超时；修复并重签后，joiner 已不再停在原 connect 超时，而是继续推进到 `getJoinedPoolView()` 路径抛出新的 `ApiError`
- `PoolController.joinByCode()` 成功后的 `getJoinedPoolView()` 分支此前未兜住 `ApiError`
  - 处理：补 `try/on ApiError`，将后端 `code/message` 映射回页面错误态与提示文案，先避免真实调试链路因未捕获异常直接中断
- 直接用同一个 `cardmind.app` 反复 `flutter run/open` 不能稳定形成真正的 macOS 双实例
  - 现状：改为复制 `cardmind-owner.app` / `cardmind-joiner.app`、分别设置 `com.example.cardmind.owner` / `com.example.cardmind.joiner` 后，joiner 已可真实成功到 `joined:<poolId>`；此前 `getJoinedPoolView()` 的 `ApiError` 高度怀疑与“伪双开 / 共享容器”有关
- iOS 模拟器容器里存在历史旧池时，`get_joined_pool_view()` 会误命中旧池
  - 现状：SQLite `pool_members` 明确显示新池已经写入当前 iOS endpoint 和 macOS endpoint，但 API 过去先取 `get_any_pool()`，导致读到旧池后对当前 endpoint 返回 `NOT_MEMBER`；修复后真实 `macOS owner -> iOS simulator joiner` 已成功 joined
- `test/widget/pages/pool_page_test.dart` 存在原有异步关停噪音
  - 现状：目标用例会在测试关停阶段报 `Cannot close sink while adding stream`，本轮未扩展处理
