## 遇到的问题

- `flutter run -d iPhone 16 Pro` 首次复验时，真实报错变为 FRB 解码 panic：`left: 316 right: 315`
  - 处理：核对时间戳后确认 iOS simulator 使用的 `rust/target/aarch64-apple-ios-sim/release/libcardmind_rust.dylib` 仍停留在前一天，晚于 Dart/Rust/FRB 源码签名变更
- iOS 工程现有 `Copy Rust Framework` build phase 只复制已有 dylib，不会自动重建
  - 处理：在 build phase 中按 `SDK_NAME` 选择 `RUST_TARGET`，先执行 `cargo build --release --target "$RUST_TARGET"`，再复制到 app bundle
- `flutter run` 触发后 `ios/Podfile.lock` 的 `PODFILE CHECKSUM` 自动变化
  - 现状：这是 `Podfile` 变更后的预期衍生产物，不是额外业务改动
