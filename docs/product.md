# 产品定义

> 面向唯一用户的个人工具。上次更新：2026-06-25

---

## 一句话

一个跨平台（Android / Windows / macOS）的纯粹 Markdown 笔记工具，给只有我自己一个人用。

---

## 痛点

- 内置备忘录不纯粹：打开看到思维导图推荐、云服务警告
- Markdown 中英文半角符号不兼容，写不了完整语法
- 没有标签系统，回头找笔记困难
- 同步依赖云服务，容量被照片占用，担心丢失

---

## 不做什么

- 不做多人协同编辑
- 不做团队协作
- 不做云端同步（只走局域网）
- 不做平台化扩张

---

## 交互原则

以微信聊天框为审美基准：输入流畅、无多余按钮、写完即走。

核心交互：
1. 打开编辑器 → 焦点自动落编辑区，不用多点一下
2. 首行即为标题，编辑器内无独立标题字段
3. 关闭页面 → 自动保存，无弹窗确认

---

## 验收条件（9 条）

基于创始人实际使用内置备忘录的痛点提炼：

```
□ 1. 从点击图标到开始打字 ≤ 2 秒
□ 2. 完整 Markdown + 中文半角符号兼容
□ 3. 写作时无多余提示/干扰
□ 4. 标签系统（不只是文件夹）
□ 5. 手机快捷入口：新建可直接写，也能直接搜索/查看已有
□ 6. 电脑端入口不比备忘录复杂
□ 7. 手机写的笔记关机前自动出现在电脑上（LAN 同步）
□ 8. 同步只走局域网，不依赖外网
□ 9. 不受云服务容量限制
```

**第一阶段交货底线（三缺二不交货）：**

```
✅ 编辑器打开即写，不用等加载
✅ 标题从第一行自动取，列表页能看到
✅ 完整 Markdown + 中文半角符号兼容
```

满足三条 → 停手，开始自用。

---

## 第一阶段验证策略

只在 Android 端验证本地体验，不做同步。自用三天后回答一个问题：

**"这个东西解决了我的痛点吗？"**

→ 回答了"是"：进入第二阶段（局域网同步）
→ 回答了"不是"：先修，再问
