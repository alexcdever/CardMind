# CardMind 用户使用指南

## 简介

CardMind 面向有多设备使用场景的个人用户，重点帮助你在不同设备之间持续记录、查看和流转自己的笔记。

本文档只描述当前已经支持的公开行为。正式的产品定位与行为约束以 `docs/specs/` 下的规格文档为准。

## 这份指南适合你做什么

- 在本地创建、编辑、查找自己的笔记
- 在设备之间延续自己的笔记使用流程
- 通过数据池完成与笔记流转相关的协作操作
- 在遇到同步异常时找到下一步可执行动作

## 快速开始

首次进入应用后，你会看到三个主要区域：

- `卡片`：查看、创建、编辑和查找笔记
- `池`：创建数据池、加入数据池、查看池内状态
- `设置`：当前版本的占位页

应用默认进入 `卡片` 页面，方便你先开始记录内容。

## 卡片使用

### 创建笔记

你可以新建一张卡片笔记，并让它出现在默认列表中。

### 编辑笔记

你可以修改已有笔记的标题和正文。保存后，列表和详情会显示更新后的内容。

### 删除笔记

当前版本支持将笔记从默认列表中移除。删除后，这条笔记不会继续显示在默认列表里。

### 查找笔记

你可以通过关键词查找笔记，搜索会同时匹配标题和正文内容。

### 按数据池查看

如果你的笔记分布在不同数据池中，可以按所属数据池筛选查看。

## 数据池使用

数据池是让笔记在个人多设备流程之外继续流转的扩展能力，但它不会替代你以个人记录为主的使用方式。

### 创建数据池

你可以创建一个新的数据池。创建完成后，创建者会成为该池的管理者。

### 加入数据池

你可以提交加入指定数据池的申请，等待处理结果。

### 管理加入申请

如果你是数据池管理者，可以处理待加入申请，也可以拒绝不合适的申请。

### 退出或解散数据池

- 普通成员可以退出数据池
- 管理者可以解散数据池
- 这些操作属于影响较大的动作，应用会先给出明确确认

### 池内笔记同步

池内成员对笔记进行创建、编辑或删除状态变化后，结果会逐步同步到其他成员可见的状态。

如果网络或同步过程出现异常，界面会给出可见提示，并提供下一步动作，例如重试或重新连接。异常处理不应阻断你继续进行本地笔记读写。

## 设置页说明

当前版本的 `设置` 页面主要作为占位区域使用。你仍然可以从这里一步返回 `卡片` 或 `池`，不会被困在额外的多级路径中。

## 常见情况

### 看不到刚删除的笔记

默认列表只显示当前可见的笔记。删除后的笔记不会继续出现在默认列表中。

### 搜索结果不完整

搜索面向当前可见的笔记，并同时匹配标题和正文。你可以检查关键词是否出现在这两个位置。

### 数据池操作失败

如果创建池、加入池、审批、退出或解散过程中失败，界面应提供明确提示，并告诉你下一步可以做什么，例如返回、取消、重试或重新连接。

### 同步出现异常

同步异常会在相关区域直接显示。你可以先根据页面提示处理异常，再继续当前操作。
