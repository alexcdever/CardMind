# CardMind UI设计规范

## 1. 设计理念

### 1.1 核心原则
- **简洁明了**: 采用简约设计风格，减少视觉干扰，让用户专注于内容创建和管理
- **直观易用**: 遵循用户直觉，提供符合常规认知的交互模式
- **一致统一**: 在不同设备和平台上保持一致的视觉和交互体验
- **响应迅速**: 确保所有操作都有即时反馈，提升用户体验
- **可访问性**: 确保应用对所有用户都友好易用
- **交互位置优化**: 交互操作尽量放在屏幕靠下位置，显示的内容包括提示内容尽量放在屏幕中间或屏幕靠上的位置

### 1.2 适用范围
本设计规范适用于CardMind应用的所有平台版本，包括桌面端（Windows、macOS、Linux）和移动端（Android、iOS）。

## 2. 色彩系统

### 2.1 主色调

| 颜色 | 色值 | 用途 |
|------|------|------|
| 主色 | `#3b82f6` (蓝色) | 品牌标识、主要操作按钮、高亮状态 |
| 辅助色 | `#8b5cf6` (紫色) | 次要操作、特殊标记 |
| 成功色 | `#10b981` (绿色) | 成功状态、完成操作 |
| 警告色 | `#f59e0b` (橙色) | 警告提示、需要注意的信息 |
| 错误色 | `#ef4444` (红色) | 错误状态、删除操作 |

### 2.2 中性色

| 颜色 | 色值 | 用途 |
|------|------|------|
| 背景色 | `#f9fafb` (浅灰) | 页面主背景 |
| 卡片背景 | `#ffffff` (白色) | 卡片、弹窗背景 |
| 文本主色 | `#111827` (深灰) | 主要文本内容 |
| 文本次要色 | `#4b5563` (中灰) | 辅助说明文本 |
| 边框色 | `#e5e7eb` (极浅灰) | 元素边框、分割线 |

## 3. 排版系统

### 3.1 文字层级

| 类型 | 字体大小 | 字重 | 颜色 | 用途 |
|------|----------|------|------|------|
| 小标题 | 16px | 600 | `#111827` | 卡片标题 |
| 正文 | 14px-16px | 400 | `#111827` | 主要内容文本 |
| 辅助文字 | 12px-14px | 400 | `#4b5563` | 描述、提示、时间戳 |

## 4. 组件设计规范

### 4.1 卡片组件

#### 4.1.1 标准卡片
- **尺寸**: 
  - 移动端：宽度100%，高度自适应
  - 桌面端：最大宽度600px，高度自适应
- **交互状态**: 
  - 正常状态：白色背景，浅灰色边框
  - 悬停状态：轻微背景色变化
  - 点击状态：明显的按压反馈

#### 4.1.2 卡片内容布局
- **标题**: 16px，加粗
- **内容**: 14px，多行文本省略
- **元信息**: 12px，次要文本颜色，位于卡片底部

### 4.2 按钮组件

#### 4.2.1 主要按钮
- **尺寸**: 高度44px
- **背景色**: 主色调
- **文字颜色**: 白色

#### 4.2.2 悬浮操作按钮 (FAB)
- **尺寸**: 56px圆形
- **背景色**: 主色调
- **位置**: 右下角固定位置
- **用途**: 用于创建新卡片

### 4.3 输入组件

#### 4.3.1 文本输入框
- **尺寸**: 高度44px
- **背景色**: 白色
- **边框**: 1px浅灰色边框

### 4.4 弹窗组件

#### 4.4.1 标准弹窗
- **尺寸**: 
  - 移动端：宽度90%，高度自适应
  - 桌面端：最大宽度500px，高度自适应
- **背景色**: 白色
- **布局**: 
  - 标题区域：顶部
  - 内容区域：中间，自适应高度
  - 按钮区域：底部，操作按钮

#### 4.4.2 确认弹窗
- **布局**: 居中显示，包含确认和取消按钮
- **按钮顺序**: 取消按钮在左，确认按钮在右

## 5. 导航设计规范

### 5.1 底部导航栏
- **高度**: 56px
- **背景色**: 白色
- **图标尺寸**: 24px
- **文字尺寸**: 12px

### 5.2 导航项规范
- **核心导航项**: 
  - 主页/卡片列表
  - 网络设置
- **间距**: 导航项之间均匀分布

## 6. 适配规范

### 6.1 平台适配原则
- **桌面端**: 优化鼠标交互，增大点击区域
- **移动端**: 优化触摸交互，适合手指操作的元素尺寸
- **跨平台**: 保持核心功能和布局一致，根据平台特性调整细节
