### 0.1 一句话摘要

本规格定义 CardMind 各分域共享的可观察行为契约，用于统一实现边界与验收口径，不包含技术实现细节。

---

## 1. 目的与适用范围

本规格定义 CardMind 分域规格的共享约束，作为分域规格之上的总则。

本规格适用于以下分域：

1. 卡片笔记（card-note）。
2. 数据池（pool）。
3. 本地设置（local-settings）。
4. UI 设计与交互（ui-interaction）。

本规格不覆盖：

1. 分域私有流程细节。
2. 具体技术实现方案。

## 2. 统一术语

1. 可观察行为：用户或外部系统可感知且可验证的行为结果。
2. 业务动作：触发领域状态变化的业务操作。
3. 错误语义：稳定且可识别的错误类别与含义。
4. 可恢复失败：可通过明确动作继续推进的失败状态。
5. 审计追溯：可将关键业务结果关联到触发该结果的业务动作。

## 3. 规范性规则

本节采用以下术语强度：

1. MUST：必须满足。
2. FORBIDDEN：禁止。
3. SHOULD：建议满足，若不满足需有明确理由。
4. MAY：可选能力。

## 4. 跨域一致性契约

1. 任何影响可观察行为的变更 MUST 先更新 `docs/specs/*`，再更新实现。
2. 重复提交同一业务动作 MUST 满足幂等预期。
3. 系统 MAY 存在短暂可见延迟，但跨参与方可观察结果 MUST 最终一致。
4. 关键业务结果 MUST 可被审计追溯到对应业务动作。
5. 本契约约束底层合并算法为 FORBIDDEN。

## 5. 跨域权限契约

1. 越权操作 MUST 返回稳定权限错误语义。
2. 越权操作静默失败为 FORBIDDEN。
3. 分域规格 MAY 细化角色，但改变共享权限处理原则为 FORBIDDEN。

## 6. 跨域错误语义契约

1. 错误码语义 MUST 稳定。
2. 用户提示文案 MAY 演进，但改变错误语义为 FORBIDDEN。
3. 可恢复失败 MUST 提供至少一个明确恢复动作。
4. 各分域依赖底层异常字符串作为错误判定依据为 FORBIDDEN。

## 7. 跨域验收契约

1. 验收场景 SHOULD 使用 Given/When/Then 表达。
2. 验收 MUST 以外部可观察行为为通过依据。
3. 验收依赖函数名、类名、表结构、日志文本等实现细节为 FORBIDDEN。
4. 分域新增行为 MUST 同步新增对应黑盒验收场景。

## 8. 与分域规格关系与演进

1. 分域规格可补充领域对象、状态机与特有语义，但违反本契约为 FORBIDDEN。
2. 若分域规格与本契约冲突，默认以本契约为准；需要例外时 MUST 先修订本契约。
3. 新增跨域语义时，SHOULD 先更新本契约，再下沉至相关分域规格。
